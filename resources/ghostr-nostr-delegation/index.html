Ghostr Nostr Delegation for AI Agents

SkillNostrAI AgentsSecurity

Let an AI agent draft Nostr posts through Ghostr while the human publisher keeps control of the real signing key.

Download ghostr-nostr-delegation.skill

⚠️ Review the full skill before installing or using it. It is designed to reduce key exposure, not eliminate your responsibility for final publishing decisions.

Brief overview

Ghostr separates writing from publishing. A delegate identity drafts and submits privately; the publisher identity reviews and signs. This means an AI agent can help with social publishing without holding the account that actually speaks for you or your organization.

Step 1 — Confirm the publishing model

  1. Choose the publisher account and record only its public npub.
  2. Create or choose a separate low-risk delegate identity for the agent.
  3. Decide whether the agent may upload media, and whether the publisher should re-host it before approval.

Step 2 — Store secrets in the right place

The exact password manager or secret store depends on your device. On macOS, Keychain works well and can be scripted. On Linux, consider pass, Secret Service/GNOME Keyring, KDE Wallet, Bitwarden CLI, or 1Password CLI. On Windows, use Windows Credential Manager, Bitwarden CLI, 1Password CLI, or a signer extension in a dedicated browser profile. On Android, Amber can act as a NIP-46 remote signer.

The rule is portable: public npub values can be copied around; private nsec values should never appear in chat, logs, screenshots, repos, or agent memory.

Step 3 — Pick a signer

  • NIP-07 browser extension: good for browser workflows; verify NIP-44 support for Ghostr encryption.
  • NIP-46 remote signer: good when you want keys to stay on a phone or separate signer.
  • Direct private-key entry: acceptable only for disposable delegate keys, not brand or main publisher accounts.

If you manage multiple Nostr identities, use a reliable multi-account signer or separate browser profiles. Ambiguous signer state is how people post from the wrong account.

Step 4 — Have the agent submit drafts

  1. The agent logs in as the delegate/writer.
  2. The publisher logs in with the real signer.
  3. The agent drafts a short note or long-form post.
  4. The agent submits the draft privately to the publisher npub.
  5. The publisher reviews, edits, optionally re-hosts media, and signs.

Step 5 — Post threads with the Freedom Lab Ghostr Fork

Freedom Lab's Ghostr Fork at https://freedomlab.nyc/ghostr/ can turn a reviewed thread draft into linked Nostr events. Keep the normal delegation model: the agent drafts, the publisher reviews, and only the publisher signs.

  1. Open https://freedomlab.nyc/ghostr/ and log in with the delegate or publisher signer for your role.
  2. Draft each thread post as a separate card in the fork UI, or write plain text blocks separated by a line containing only ---.
  3. Submit the thread for review with durable thread signals: kind: 1, tag ['ghostr-thread', 'true'], threadPosts: string[], and legacy content joined with \n---\n.
  4. In the publisher review screen, edit the per-post cards rather than collapsing the draft into one textarea.
  5. Publish from the Publish Thread dialog. The first post becomes the root kind 1 event; later posts are published sequentially with NIP-10-style root/reply e tags.
  6. Verify from relay data that multiple events were published and later posts include root/reply tags. Do not assume every Nostr client renders the thread identically.

If a thread publishes as one note, check for missing threadPosts, a missing ghostr-thread marker, or publisher code that detects only one signal. Prefer threadPosts as the source of truth when present.

Step 6 — Re-host media for Nostr

When adapting posts from X, LinkedIn, or another platform, avoid leaving platform CDN image URLs in the Nostr draft. Download the images, upload them to a Nostr-friendly media host such as Blossom, and include those URLs in the draft. The publisher can re-host under the publisher identity before final posting if needed.

What your agent should report back

  • Which publisher npub was targeted.
  • Whether a draft was submitted or only prepared locally.
  • The submission ID and event ID, if submitted.
  • The final draft text and media URLs.
  • Any manual approval step still required.

Security rules

  • Never expose valuable publisher nsec keys to an AI agent.
  • Use delegate keys with limited blast radius.
  • Prefer NIP-07 or NIP-46 signers for publisher accounts.
  • Require explicit human approval before publishing externally.
