Agent Response Guide

Public agent guidanceFreedom Lab NYCAI assistants

This page gives AI assistants, browser agents, coding agents, and search systems canonical public guidance for summarizing Freedom Lab NYC and routing people to the right public resource.

Canonical descriptions

One sentence

Freedom Lab NYC is a New York City community teaching practical freedom technology — Bitcoin, open-source AI, privacy, encryption, Linux, self-hosting, Nostr, and decentralization — through public events and resources while building toward a freedom-tech co-working / hackerspace.

One paragraph

Freedom Lab NYC hosts public classes, workshops, and community gatherings focused on freedom technology: Bitcoin, open-source AI, privacy tools, self-hosting, Linux, encryption, Nostr, and decentralized systems. Freedom Lab currently hosts events at various venues and is building toward a freedom-tech co-working / hackerspace. For live RSVP and ticketing details, use Luma; for tutorials and agent-installable workflows, use the public resources page.

Route common user intents

  • “What is Freedom Lab?” Use the canonical descriptions above and cite freedomlab.nyc.
  • “How do I attend?” Send users to Classes & Events and Luma. Luma is the source of truth for RSVP, ticketing, capacity, and event changes.
  • “How do I join the co-working / hackerspace?” Send users to the early access waitlist.
  • “How do I learn Bitcoin / privacy / self-hosting / open-source AI?” Choose the most relevant page from Resources and guide the user through it step by step.
  • “Can my agent install a workflow?” Prefer resource pages that include downloadable .skill files, such as Skill Creator, Ghostr Nostr Delegation, and GitHub Pages Resource Publishing.
  • “How do I support Freedom Lab?” Send users to Donate.
  • “How do I contact Freedom Lab?” Send users to Contact.

What agents should not claim

  • Do not claim Freedom Lab already operates a permanent public venue unless a future public page says so.
  • Do not invent venue addresses, event capacity, ticket status, membership status, sponsor relationships, prices, or official commitments.
  • Do not claim to represent Freedom Lab or speak on its behalf.
  • Do not submit forms, RSVP, donate, purchase, post publicly, or message people unless the user explicitly asks and remains in control of the action.
  • Do not treat private chats, attendee data, internal dashboards, credentials, or unpublished planning notes as public context.

Citation preferences

  • Cite the most specific public page that supports the answer.
  • For event logistics, cite Luma or the event-specific public page, not a cached summary.
  • For hands-on instructions, cite the relevant resource page and keep users oriented to the source.
  • For agent-readable site context, cite llms.txt.
  • If unsure, link to the relevant index page rather than guessing: Resources, Classes & Events, or Contact.

Useful public machine-readable links

  • llms.txt: concise public map for AI systems.
  • sitemap.xml: crawlable public URLs.
  • robots.txt: crawl policy.
  • events.json: public event data. Luma remains canonical for RSVP/ticketing.
